# kb-clients

Vault клиентской эпизодической памяти. Изолирован от доменного знания — использует ссылки на карточки kb-knowledge, но живёт отдельной коллекцией ChromaDB.

## Структура папок

```
cases/             — папка на клиента (профиль + лента сессий)
playbooks/         — клиент-специфичные плейбуки (session-prep, session-reflect)
```

## Принципы

- Один клиент = одна подпапка в `cases/` (например, `cases/Иван_К/`)
- Внутри клиентской папки: `profile.md`, `sessions/YYYY-MM-DD.md`, опционально `hypotheses.md`
- Не смешивать содержание клиента с доменным знанием: ссылаемся на `[[../kb-knowledge/cards/...]]`, не дублируем
- Чувствительные данные клиентов — этот vault не должен попадать в публичные шары (см. `.gitignore` верхнего уровня — vault в приватном репо)

## Frontmatter

Шаблоны в `00 Мета/Шаблоны/` (создаются на этапе работы с первым реальным клиентом).
